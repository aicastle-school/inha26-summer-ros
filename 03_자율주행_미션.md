# LIMO Ackermann 센서 수신 및 `/cmd_vel` 제어 예제

이 문서는 **LIMO 차량**을 기준으로, ROS2 Humble 환경에서 LiDAR, IMU, Camera, Odom 데이터를 받아와 `/cmd_vel`로 제어하는 실습 예제이다.

기준 조건은 다음과 같다.

```text
차량: LIMO
주행 모드: Ackermann 모드
ROS 버전: ROS2 Humble
LiDAR 토픽: /scan
IMU 토픽: /imu
Camera 토픽: /camera/color/image_raw/compressed
제어 토픽: /cmd_vel
Odom 토픽: /odom
```

Ackermann 차량은 자동차처럼 조향하면서 움직인다. 따라서 이 문서에서는 **제자리 회전 예제는 사용하지 않는다.** 모든 회전 예제는 `linear.x`로 전진하면서 `angular.z`로 조향하는 방식으로 작성한다.

또한 LIMO Ackermann 모드에서 조향 명령은 다음 범위로 제한한다.

```text
angular.z 최소값: -0.42
angular.z 최대값:  0.42
```

모든 예제 코드에는 아래 제한 함수를 넣어 `angular.z`가 반드시 이 범위를 넘지 않게 한다.

```python
MAX_STEER = 0.42

def clamp_steer(value):
    return max(-MAX_STEER, min(MAX_STEER, value))
```

## 0. 토픽과 메시지 타입 설명

| 역할 | 토픽 | 메시지 타입 | 설명 |
|---|---|---|---|
| LiDAR | `/scan` | `sensor_msgs/msg/LaserScan` | 각도별 거리 배열을 제공한다. `ranges` 배열에 거리값이 들어 있다. |
| IMU | `/imu` | `sensor_msgs/msg/Imu` | 자세, 각속도, 선형가속도 데이터를 제공한다. 자세는 quaternion 형식이다. |
| Camera | `/camera/color/image_raw/compressed` | `sensor_msgs/msg/CompressedImage` | 압축된 카메라 이미지를 제공한다. OpenCV에서 `cv2.imdecode()`로 복원한다. |
| Control | `/cmd_vel` | `geometry_msgs/msg/Twist` | 로봇 속도 명령이다. LIMO Ackermann에서는 `linear.x`와 `angular.z`를 주로 사용한다. |
| Odom | `/odom` | `nav_msgs/msg/Odometry` | 로봇의 추정 위치, 자세, 선속도, 각속도 정보를 제공한다. |

### `/scan` 메시지 핵심 필드

```text
angle_min       : 첫 번째 거리값의 각도
angle_max       : 마지막 거리값의 각도
angle_increment : ranges 배열에서 index가 1 증가할 때 증가하는 각도
ranges          : 각도별 거리값 배열
```

`ranges[i]`가 몇 도 방향인지 계산하려면 다음 식을 사용한다.

```python
angle_rad = msg.angle_min + i * msg.angle_increment
```

### `/imu` 메시지 핵심 필드

```text
orientation          : quaternion 자세값
angular_velocity     : 각속도
linear_acceleration  : 선형가속도
```

IMU 자세는 quaternion이라서 사람이 바로 이해하기 어렵다. 그래서 이 문서에서는 `tf_transformations` 라이브러리로 roll, pitch, yaw를 만든다.

```python
from tf_transformations import euler_from_quaternion
```

### `/camera/color/image_raw/compressed` 메시지 핵심 필드

```text
format : 압축 이미지 형식
 data  : 압축된 이미지 바이트 데이터
```

OpenCV로 보려면 다음 흐름이 필요하다.

```text
CompressedImage 수신
→ numpy 배열로 변환
→ cv2.imdecode()로 이미지 복원
→ 전처리
→ cv2.imshow()로 실시간 확인
```

### `/cmd_vel` 메시지 핵심 필드

```text
linear.x  : 전진/후진 속도
angular.z : 조향 방향에 해당하는 회전 명령
```

LIMO Ackermann 예제에서는 다음 값만 사용한다.

```text
linear.x
angular.z
```

그리고 `angular.z`는 반드시 다음 범위로 제한한다.

```text
-0.42 <= angular.z <= 0.42
```

### `/odom` 메시지 핵심 필드

```text
pose.pose.position.x       : x 위치
pose.pose.position.y       : y 위치
pose.pose.orientation      : quaternion 자세
 twist.twist.linear.x      : 현재 전진 속도
 twist.twist.angular.z     : 현재 회전 속도
```

## 2.1 LiDAR 예제 1: 전방 장애물이 가까우면 정지

### 문제의 의의

Ackermann 차량은 제자리에서 바로 피하기 어렵다. 그래서 가장 기본적인 안전 제어는 **전방 장애물을 감지하면 즉시 정지**하는 것이다.

### 코드 흐름

```text
/scan 수신
→ ranges 가운데 값을 전방 거리로 사용
→ 0.5m보다 가까우면 정지
→ 아니면 천천히 전진
→ /cmd_vel 발행
```

파일명: `sensor_cmd_examples/lidar_obstacle_stop.py`

```python
import rclpy  # ROS2 Python 라이브러리를 가져온다.
from rclpy.node import Node  # ROS2 노드 클래스를 가져온다.
from sensor_msgs.msg import LaserScan  # /scan 메시지 타입을 가져온다.
from geometry_msgs.msg import Twist  # /cmd_vel 메시지 타입을 가져온다.

MAX_STEER = 0.42  # LIMO Ackermann 모드에서 사용할 angular.z 제한값이다.


def clamp_steer(value):  # 조향 명령이 제한값을 넘지 않게 만드는 함수이다.
    return max(-MAX_STEER, min(MAX_STEER, value))  # -0.42 ~ 0.42 사이로 값을 제한한다.


class LidarObstacleStop(Node):  # LiDAR 장애물 정지 노드이다.
    def __init__(self):  # 노드가 생성될 때 실행된다.
        super().__init__('lidar_obstacle_stop')  # 노드 이름을 설정한다.
        self.cmd_pub = self.create_publisher(Twist, '/cmd_vel', 10)  # 제어 명령 Publisher를 만든다.
        self.scan_sub = self.create_subscription(LaserScan, '/scan', self.scan_callback, 10)  # /scan을 구독한다.

    def scan_callback(self, msg):  # LiDAR 데이터가 들어올 때마다 실행된다.
        front_index = len(msg.ranges) // 2  # ranges 배열의 가운데를 전방으로 가정한다.
        front_distance = msg.ranges[front_index]  # 전방 거리값을 가져온다.

        cmd = Twist()  # 속도 명령 메시지를 만든다.

        if front_distance < 0.5:  # 전방 장애물이 0.5m보다 가까우면 위험하다.
            cmd.linear.x = 0.0  # 차량을 정지시킨다.
            cmd.angular.z = clamp_steer(0.0)  # 조향도 0으로 둔다.
            self.get_logger().warn('Obstacle detected. Stop.')  # 정지 상태를 출력한다.
        else:  # 전방이 비어 있으면 주행 가능하다.
            cmd.linear.x = 0.2  # 천천히 전진한다.
            cmd.angular.z = clamp_steer(0.0)  # 직진 조향을 유지한다.
            self.get_logger().info('Path clear. Move forward.')  # 전진 상태를 출력한다.

        self.cmd_pub.publish(cmd)  # /cmd_vel 명령을 발행한다.


def main(args=None):  # 실행 시작 함수이다.
    rclpy.init(args=args)  # ROS2를 초기화한다.
    node = LidarObstacleStop()  # 노드를 만든다.
    rclpy.spin(node)  # 콜백을 계속 기다린다.
    node.destroy_node()  # 노드를 정리한다.
    rclpy.shutdown()  # ROS2를 종료한다.
```

## 2.2 LiDAR 예제 2: `enumerate()`로 각도 계산 후 전방 장애물 정지

### 문제의 의의

LiDAR의 `ranges` 배열은 단순한 거리 목록이다. 실제 제어에서는 **각 index가 몇 도 방향인지 계산**해야 한다. 이 예제는 `enumerate()`로 index와 거리값을 같이 읽고, 특정 각도 구간만 검사한다.

### 코드 흐름

```text
/scan 수신
→ enumerate(msg.ranges)로 index와 거리값을 함께 읽음
→ angle_min + index * angle_increment로 각도 계산
→ -30도 ~ +30도 전방 구간만 검사
→ 가까운 장애물이 있으면 정지
```

파일명: `sensor_cmd_examples/lidar_angle_stop.py`

```python
import math  # 라디안과 각도 변환을 위해 사용한다.
import rclpy  # ROS2 Python 라이브러리를 가져온다.
from rclpy.node import Node  # ROS2 노드 클래스를 가져온다.
from sensor_msgs.msg import LaserScan  # /scan 메시지 타입을 가져온다.
from geometry_msgs.msg import Twist  # /cmd_vel 메시지 타입을 가져온다.

MAX_STEER = 0.42  # LIMO Ackermann 조향 제한값이다.


def clamp_steer(value):  # 조향 제한 함수이다.
    return max(-MAX_STEER, min(MAX_STEER, value))  # 값을 -0.42 ~ 0.42로 제한한다.


class LidarAngleStop(Node):  # 각도 기반 장애물 정지 노드이다.
    def __init__(self):  # 노드 생성 시 실행된다.
        super().__init__('lidar_angle_stop')  # 노드 이름을 설정한다.
        self.cmd_pub = self.create_publisher(Twist, '/cmd_vel', 10)  # /cmd_vel Publisher를 만든다.
        self.scan_sub = self.create_subscription(LaserScan, '/scan', self.scan_callback, 10)  # /scan을 구독한다.

    def scan_callback(self, msg):  # LiDAR 메시지가 들어올 때마다 실행된다.
        obstacle_found = False  # 전방 장애물 발견 여부를 저장한다.

        for index, distance in enumerate(msg.ranges):  # index와 거리값을 동시에 꺼낸다.
            angle_rad = msg.angle_min + index * msg.angle_increment  # 현재 index의 라디안 각도를 계산한다.
            angle_deg = math.degrees(angle_rad)  # 사람이 보기 쉽게 degree로 변환한다.

            if -30.0 <= angle_deg <= 30.0:  # 차량 전방 -30도 ~ +30도 범위만 검사한다.
                if distance < 0.5:  # 해당 방향에 0.5m보다 가까운 장애물이 있으면 위험하다.
                    obstacle_found = True  # 장애물 발견 상태로 바꾼다.
                    break  # 더 검사할 필요가 없으므로 반복문을 끝낸다.

        cmd = Twist()  # 속도 명령 메시지를 만든다.

        if obstacle_found:  # 전방 구간에서 장애물을 찾았다면 정지한다.
            cmd.linear.x = 0.0  # 전진 속도를 0으로 만든다.
            cmd.angular.z = clamp_steer(0.0)  # 조향 명령도 0으로 만든다.
            self.get_logger().warn('Obstacle in front angle range. Stop.')  # 경고를 출력한다.
        else:  # 전방 구간이 안전하다면 전진한다.
            cmd.linear.x = 0.2  # 천천히 전진한다.
            cmd.angular.z = clamp_steer(0.0)  # 직진한다.
            self.get_logger().info('Front angle range is clear.')  # 상태를 출력한다.

        self.cmd_pub.publish(cmd)  # 제어 명령을 발행한다.


def main(args=None):  # 실행 시작 함수이다.
    rclpy.init(args=args)  # ROS2를 초기화한다.
    node = LidarAngleStop()  # 노드를 생성한다.
    rclpy.spin(node)  # 콜백 대기 상태로 둔다.
    node.destroy_node()  # 노드를 정리한다.
    rclpy.shutdown()  # ROS2를 종료한다.
```

## 2.3 LiDAR 예제 3: 좌우 벽 거리 차이로 복도 중앙 주행

### 문제의 의의

복도나 온실 고랑처럼 양쪽 경계가 있는 환경에서는 좌우 거리 차이를 이용해 중앙을 유지할 수 있다. 이 예제는 LiDAR를 단순 장애물 센서가 아니라 **조향 보정 센서**로 사용하는 예제이다.

### 코드 흐름

```text
/scan 수신
→ 왼쪽 60도 거리와 오른쪽 -60도 거리 추출
→ 좌우 거리 차이 계산
→ 차이가 크면 가까운 벽 반대쪽으로 조향
→ angular.z를 -0.42 ~ 0.42로 제한
```

파일명: `sensor_cmd_examples/lidar_corridor_follow.py`

```python
import math  # 각도 계산에 사용한다.
import rclpy  # ROS2 Python 라이브러리이다.
from rclpy.node import Node  # 노드 클래스를 가져온다.
from sensor_msgs.msg import LaserScan  # LaserScan 메시지를 가져온다.
from geometry_msgs.msg import Twist  # Twist 메시지를 가져온다.

MAX_STEER = 0.42  # LIMO Ackermann angular.z 제한값이다.


def clamp_steer(value):  # 조향 명령 제한 함수이다.
    return max(-MAX_STEER, min(MAX_STEER, value))  # -0.42 ~ 0.42 범위로 제한한다.


class LidarCorridorFollow(Node):  # 복도 중앙 주행 노드이다.
    def __init__(self):  # 노드 생성 시 실행된다.
        super().__init__('lidar_corridor_follow')  # 노드 이름을 설정한다.
        self.cmd_pub = self.create_publisher(Twist, '/cmd_vel', 10)  # 제어 명령 Publisher를 만든다.
        self.scan_sub = self.create_subscription(LaserScan, '/scan', self.scan_callback, 10)  # /scan을 구독한다.

    def get_distance_at_angle(self, msg, target_deg):  # 원하는 각도의 거리값을 꺼내는 함수이다.
        target_rad = math.radians(target_deg)  # degree를 radian으로 바꾼다.
        index = int((target_rad - msg.angle_min) / msg.angle_increment)  # 각도에 해당하는 index를 계산한다.
        index = max(0, min(len(msg.ranges) - 1, index))  # index가 배열 범위를 벗어나지 않게 제한한다.
        return msg.ranges[index]  # 해당 각도의 거리값을 반환한다.

    def scan_callback(self, msg):  # LiDAR 데이터가 들어올 때 실행된다.
        left_distance = self.get_distance_at_angle(msg, 60.0)  # 왼쪽 60도 방향 거리이다.
        right_distance = self.get_distance_at_angle(msg, -60.0)  # 오른쪽 -60도 방향 거리이다.

        error = left_distance - right_distance  # 양수면 왼쪽 공간이 더 넓다는 뜻이다.
        steer_cmd = 0.5 * error  # 거리 차이를 조향 명령으로 바꾼다.
        steer_cmd = clamp_steer(steer_cmd)  # LIMO 조향 제한값을 적용한다.

        cmd = Twist()  # 속도 명령 메시지를 만든다.
        cmd.linear.x = 0.15  # 천천히 전진한다.
        cmd.angular.z = steer_cmd  # 제한된 조향 명령을 넣는다.

        self.cmd_pub.publish(cmd)  # /cmd_vel을 발행한다.
```

## 3.1 IMU 예제 1: 현재 yaw 유지하며 직진 보정

### 문제의 의의

Ackermann 차량이 직진 명령을 받아도 바닥 마찰, 바퀴 정렬, 외란 때문에 방향이 틀어질 수 있다. IMU의 yaw를 사용하면 **처음 바라본 방향을 유지하도록 조향 보정**할 수 있다.

### 코드 흐름

```text
/imu 수신
→ quaternion을 tf_transformations로 roll, pitch, yaw 변환
→ 첫 yaw를 목표 yaw로 저장
→ 현재 yaw와 목표 yaw의 오차 계산
→ 오차에 비례해서 angular.z 생성
→ -0.42 ~ 0.42 제한 후 /cmd_vel 발행
```

파일명: `sensor_cmd_examples/imu_yaw_hold.py`

```python
import math  # 각도 계산에 사용한다.
import rclpy  # ROS2 Python 라이브러리이다.
from rclpy.node import Node  # 노드 클래스를 가져온다.
from sensor_msgs.msg import Imu  # /imu 메시지 타입을 가져온다.
from geometry_msgs.msg import Twist  # /cmd_vel 메시지 타입을 가져온다.
from tf_transformations import euler_from_quaternion  # quaternion을 roll, pitch, yaw로 바꾸는 함수이다.

MAX_STEER = 0.42  # LIMO Ackermann angular.z 제한값이다.


def clamp_steer(value):  # 조향 제한 함수이다.
    return max(-MAX_STEER, min(MAX_STEER, value))  # -0.42 ~ 0.42로 제한한다.


def normalize_angle(angle):  # 각도 오차를 -pi ~ pi 범위로 정리하는 함수이다.
    return math.atan2(math.sin(angle), math.cos(angle))  # wrap-around 문제를 해결한다.


class ImuYawHold(Node):  # IMU yaw 유지 주행 노드이다.
    def __init__(self):  # 노드 생성 시 실행된다.
        super().__init__('imu_yaw_hold')  # 노드 이름을 설정한다.
        self.target_yaw = None  # 처음 yaw를 저장할 변수이다.
        self.cmd_pub = self.create_publisher(Twist, '/cmd_vel', 10)  # 제어 명령 Publisher를 만든다.
        self.imu_sub = self.create_subscription(Imu, '/imu', self.imu_callback, 10)  # /imu를 구독한다.

    def imu_callback(self, msg):  # IMU 데이터가 들어올 때마다 실행된다.
        q = msg.orientation  # IMU orientation quaternion을 꺼낸다.
        quat = [q.x, q.y, q.z, q.w]  # tf_transformations가 요구하는 리스트 형식으로 만든다.
        roll, pitch, yaw = euler_from_quaternion(quat)  # quaternion을 roll, pitch, yaw로 변환한다.

        if self.target_yaw is None:  # 아직 기준 yaw가 없다면 첫 yaw를 기준으로 잡는다.
            self.target_yaw = yaw  # 현재 yaw를 목표 yaw로 저장한다.

        error = normalize_angle(self.target_yaw - yaw)  # 목표 yaw와 현재 yaw의 차이를 계산한다.
        steer_cmd = 1.0 * error  # yaw 오차를 조향 명령으로 변환한다.
        steer_cmd = clamp_steer(steer_cmd)  # LIMO 제한 범위를 적용한다.

        cmd = Twist()  # 속도 명령 메시지를 만든다.
        cmd.linear.x = 0.15  # 천천히 전진한다.
        cmd.angular.z = steer_cmd  # 방향을 유지하기 위한 조향 명령을 넣는다.

        self.cmd_pub.publish(cmd)  # /cmd_vel을 발행한다.
        self.get_logger().info(f'yaw={math.degrees(yaw):.1f}, error={math.degrees(error):.1f}, steer={steer_cmd:.2f}')  # 상태를 출력한다.


def main(args=None):  # 실행 시작 함수이다.
    rclpy.init(args=args)  # ROS2를 초기화한다.
    node = ImuYawHold()  # 노드를 생성한다.
    rclpy.spin(node)  # 콜백을 계속 처리한다.
    node.destroy_node()  # 노드를 정리한다.
    rclpy.shutdown()  # ROS2를 종료한다.
```

## 3.2 IMU 예제 2: 현재 각도 기준 90도 Ackermann 원호 회전

### 문제의 의의

LIMO Ackermann 모드에서는 제자리 회전이 아니라 **앞으로 움직이면서 원호를 그리듯 방향을 바꾸는 방식**이 자연스럽다. 이 예제는 현재 yaw에서 90도 더한 값을 목표 방향으로 잡고, 목표 방향에 도달하면 정지한다.

### 코드 흐름

```text
/imu 수신
→ 현재 yaw 계산
→ 처음 yaw + 90도를 목표 yaw로 설정
→ 목표 yaw와 현재 yaw 오차 계산
→ 전진하면서 angular.z로 조향
→ 오차가 3도 이하면 정지
```

파일명: `sensor_cmd_examples/imu_turn_90_ackermann.py`

```python
import math  # 각도 계산에 사용한다.
import rclpy  # ROS2 Python 라이브러리이다.
from rclpy.node import Node  # 노드 클래스를 가져온다.
from sensor_msgs.msg import Imu  # /imu 메시지 타입이다.
from geometry_msgs.msg import Twist  # /cmd_vel 메시지 타입이다.
from tf_transformations import euler_from_quaternion  # quaternion을 오일러 각도로 변환한다.

MAX_STEER = 0.42  # LIMO Ackermann angular.z 제한값이다.


def clamp_steer(value):  # 조향 제한 함수이다.
    return max(-MAX_STEER, min(MAX_STEER, value))  # -0.42 ~ 0.42 범위로 제한한다.


def normalize_angle(angle):  # 각도 오차를 -pi ~ pi로 정리한다.
    return math.atan2(math.sin(angle), math.cos(angle))  # 180도 경계 문제를 해결한다.


class ImuTurn90Ackermann(Node):  # IMU 기반 90도 원호 회전 노드이다.
    def __init__(self):  # 노드 생성 시 실행된다.
        super().__init__('imu_turn_90_ackermann')  # 노드 이름을 설정한다.
        self.target_yaw = None  # 목표 yaw를 저장할 변수이다.
        self.finished = False  # 회전 완료 여부를 저장한다.
        self.cmd_pub = self.create_publisher(Twist, '/cmd_vel', 10)  # /cmd_vel Publisher를 만든다.
        self.imu_sub = self.create_subscription(Imu, '/imu', self.imu_callback, 10)  # /imu를 구독한다.

    def imu_callback(self, msg):  # IMU 데이터가 들어올 때마다 실행된다.
        q = msg.orientation  # quaternion 자세값을 꺼낸다.
        quat = [q.x, q.y, q.z, q.w]  # 변환 함수 입력 형식으로 만든다.
        roll, pitch, yaw = euler_from_quaternion(quat)  # roll, pitch, yaw로 변환한다.

        if self.target_yaw is None:  # 목표 각도가 아직 없으면 초기화한다.
            self.target_yaw = normalize_angle(yaw + math.radians(90.0))  # 현재 yaw에서 90도를 더해 목표로 잡는다.

        error = normalize_angle(self.target_yaw - yaw)  # 목표 yaw와 현재 yaw의 오차를 계산한다.
        cmd = Twist()  # 속도 명령 메시지를 만든다.

        if abs(math.degrees(error)) < 3.0:  # 목표 각도와 3도 이내로 가까워졌다면 완료로 본다.
            cmd.linear.x = 0.0  # 차량을 정지시킨다.
            cmd.angular.z = clamp_steer(0.0)  # 조향 명령도 0으로 둔다.
            self.finished = True  # 완료 상태를 저장한다.
            self.get_logger().info('Turn finished. Stop.')  # 완료 메시지를 출력한다.
        elif not self.finished:  # 아직 목표 각도에 도달하지 않았다면 계속 회전한다.
            cmd.linear.x = 0.12  # Ackermann 차량이 원호를 그리도록 전진 속도를 준다.
            cmd.angular.z = clamp_steer(1.2 * error)  # 오차에 비례한 조향 명령을 제한 범위 안에서 넣는다.
            self.get_logger().info(f'error={math.degrees(error):.1f}, steer={cmd.angular.z:.2f}')  # 현재 오차와 조향값을 출력한다.

        self.cmd_pub.publish(cmd)  # /cmd_vel을 발행한다.


def main(args=None):  # 실행 시작 함수이다.
    rclpy.init(args=args)  # ROS2를 초기화한다.
    node = ImuTurn90Ackermann()  # 노드를 생성한다.
    rclpy.spin(node)  # 콜백을 계속 처리한다.
    node.destroy_node()  # 노드를 정리한다.
    rclpy.shutdown()  # ROS2를 종료한다.
```

## 3.3 IMU 예제 3: roll/pitch 기울기 안전 정지

### 문제의 의의

실제 차량이 경사면이나 장애물에 걸리면 roll 또는 pitch가 커질 수 있다. 이 예제는 IMU를 이용해 차량 기울기가 위험해졌을 때 주행을 멈추는 안전 예제이다.

### 코드 흐름

```text
/imu 수신
→ quaternion을 roll, pitch, yaw로 변환
→ roll 또는 pitch가 15도 이상인지 확인
→ 위험하면 정지
→ 안전하면 천천히 전진
```

파일명: `sensor_cmd_examples/imu_tilt_safety_stop.py`

```python
import math  # 각도 변환에 사용한다.
import rclpy  # ROS2 Python 라이브러리이다.
from rclpy.node import Node  # 노드 클래스를 가져온다.
from sensor_msgs.msg import Imu  # /imu 메시지 타입이다.
from geometry_msgs.msg import Twist  # /cmd_vel 메시지 타입이다.
```

## 4.1 Camera 예제 1: 어두운 화면이면 정지

### 문제의 의의

카메라 기반 주행에서는 영상이 너무 어두우면 라인, 사람, 장애물 인식이 불안정해진다. 이 예제는 카메라 데이터를 받아 전처리 결과를 화면에 띄우고, 밝기가 낮으면 정지한다.

### 코드 흐름

```text
CompressedImage 수신
→ cv2.imdecode()로 이미지 복원
→ grayscale 변환
→ 평균 밝기 계산
→ cv2.imshow()로 현재 영상 확인
→ 어두우면 정지, 밝으면 전진
```

파일명: `sensor_cmd_examples/camera_brightness_stop.py`

```python
import cv2  # OpenCV 라이브러리이다.
import numpy as np  # 이미지 바이트 데이터를 배열로 바꾸기 위해 사용한다.
import rclpy  # ROS2 Python 라이브러리이다.
from rclpy.node import Node  # 노드 클래스를 가져온다.
from sensor_msgs.msg import CompressedImage  # 압축 이미지 메시지 타입이다.
from geometry_msgs.msg import Twist  # /cmd_vel 메시지 타입이다.

MAX_STEER = 0.42  # LIMO Ackermann angular.z 제한값이다.


def clamp_steer(value):  # 조향 제한 함수이다.
    return max(-MAX_STEER, min(MAX_STEER, value))  # -0.42 ~ 0.42 범위로 제한한다.


class CameraBrightnessStop(Node):  # 밝기 기반 정지 노드이다.
    def __init__(self):  # 노드 생성 시 실행된다.
        super().__init__('camera_brightness_stop')  # 노드 이름을 설정한다.
        self.cmd_pub = self.create_publisher(Twist, '/cmd_vel', 10)  # /cmd_vel Publisher를 만든다.
        self.image_sub = self.create_subscription(CompressedImage, '/camera/color/image_raw/compressed', self.image_callback, 10)  # 압축 카메라 토픽을 구독한다.

    def image_callback(self, msg):  # 카메라 이미지가 들어올 때마다 실행된다.

            return  # 콜백을 종료한다.

        cmd = Twist()  # 속도 명령 메시지를 만든다.

        if brightness < 50.0:  # 평균 밝기가 너무 낮으면 위험하다.
            cmd.linear.x = 0.0  # 정지한다.
            cmd.angular.z = clamp_steer(0.0)  # 조향도 0으로 둔다.
            self.get_logger().warn('Image is too dark. Stop.')  # 경고를 출력한다.
        else:  # 밝기가 충분하면 주행한다.
            cmd.linear.x = 0.12  # 천천히 전진한다.
            cmd.angular.z = clamp_steer(0.0)  # 직진한다.
            self.get_logger().info('Brightness is OK. Move forward.')  # 상태를 출력한다.

        self.cmd_pub.publish(cmd)  # /cmd_vel을 발행한다.

    def destroy_node(self):  # 노드가 종료될 때 실행된다.
        cv2.destroyAllWindows()  # OpenCV 창을 모두 닫는다.
        super().destroy_node()  # ROS2 노드 정리 함수를 호출한다.


def main(args=None):  # 실행 시작 함수이다.
    rclpy.init(args=args)  # ROS2를 초기화한다.
    node = CameraBrightnessStop()  # 노드를 생성한다.
    rclpy.spin(node)  # 콜백을 계속 처리한다.
    node.destroy_node()  # 노드를 정리한다.
```

## 4.2 Camera 예제 2: 검은 라인 중심 따라가기

### 문제의 의의

라인 트레이싱은 카메라 기반 로봇 제어의 가장 기본 예제이다. 이 예제는 검은 라인의 중심을 찾고, 화면 중심과 라인 중심의 차이를 `angular.z` 조향 명령으로 바꾼다.

### 코드 흐름

```text
CompressedImage 수신
→ 이미지 복원
→ grayscale 변환
→ threshold로 검은 라인 mask 생성
→ mask의 중심 x 좌표 계산
→ 화면 중심과 라인 중심 오차 계산
→ angular.z 제한 적용
→ 전진하면서 조향
→ cv2.imshow()로 mask와 중심선 확인
```

```python
import cv2  # OpenCV 라이브러리이다.
import numpy as np  # 이미지 배열 처리를 위해 사용한다.
import rclpy  # ROS2 Python 라이브러리이다.
from rclpy.node import Node  # 노드 클래스를 가져온다.
from sensor_msgs.msg import CompressedImage  # 압축 이미지 메시지 타입이다.
from geometry_msgs.msg import Twist  # /cmd_vel 메시지 타입이다.

MAX_STEER = 0.42  # LIMO Ackermann angular.z 제한값이다.


def clamp_steer(value):  # 조향 제한 함수이다.
    return max(-MAX_STEER, min(MAX_STEER, value))  # -0.42 ~ 0.42 범위로 제한한다.


class CameraLineFollow(Node):  # 카메라 라인 추종 노드이다.
    def __init__(self):  # 노드 생성 시 실행된다.
        super().__init__('camera_line_follow')  # 노드 이름을 설정한다.
        self.cmd_pub = self.create_publisher(Twist, '/cmd_vel', 10)  # /cmd_vel Publisher를 만든다.
        self.image_sub = self.create_subscription(CompressedImage, '/camera/color/image_raw/compressed', self.image_callback, 10)  # 압축 이미지 토픽을 구독한다.

    def image_callback(self, msg):  # 이미지가 들어올 때마다 실행된다.
        np_arr = np.frombuffer(msg.data, np.uint8)  # 압축 이미지 바이트를 numpy 배열로 바꾼다.
        frame = cv2.imdecode(np_arr, cv2.IMREAD_COLOR)  # 압축 이미지를 OpenCV 이미지로 복원한다.

        if frame is None:  # 이미지 복원 실패 시 종료한다.
            return  # 콜백을 끝낸다.

        height, width, _ = frame.shape  # 이미지 높이와 너비를 가져온다.
        roi = frame[int(height * 0.6):height, :]  # 바닥 라인만 보기 위해 아래쪽 영역만 사용한다.
        gray = cv2.cvtColor(roi, cv2.COLOR_BGR2GRAY)  # 흑백 이미지로 변환한다.
        _, mask = cv2.threshold(gray, 80, 255, cv2.THRESH_BINARY_INV)  # 어두운 검은 라인을 흰색 mask로 만든다.

        moments = cv2.moments(mask)  # mask의 무게중심 계산에 필요한 moment를 구한다.
        cmd = Twist()  # 속도 명령 메시지를 만든다.

        if moments['m00'] > 0:  # 라인 영역이 존재하면 중심을 계산할 수 있다.
            cx = int(moments['m10'] / moments['m00'])  # 라인의 x 중심 좌표를 계산한다.
            center_x = width // 2  # 화면의 x 중심 좌표를 계산한다.
            error = center_x - cx  # 화면 중심과 라인 중심의 차이를 계산한다.
            normalized_error = error / center_x  # 오차를 -1 ~ 1 근처 값으로 정규화한다.
            steer_cmd = clamp_steer(0.5 * normalized_error)  # 오차를 조향 명령으로 만들고 제한한다.

            cmd.linear.x = 0.12  # 전진 속도를 준다.
            cmd.angular.z = steer_cmd  # 제한된 조향 명령을 넣는다.

            cv2.circle(roi, (cx, roi.shape[0] // 2), 8, (0, 0, 255), -1)  # 라인 중심점을 표시한다.
            cv2.line(roi, (center_x, 0), (center_x, roi.shape[0]), (255, 0, 0), 2)  # 화면 중심선을 표시한다.
            self.get_logger().info(f'line cx={cx}, error={error}, steer={steer_cmd:.2f}')  # 상태를 출력한다.
        else:  # 라인을 찾지 못하면 안전하게 정지한다.
            cmd.linear.x = 0.0  # 정지한다.
            cmd.angular.z = clamp_steer(0.0)  # 조향도 0으로 둔다.
            self.get_logger().warn('Line not found. Stop.')  # 경고를 출력한다.

        cv2.imshow('line_roi', roi)  # 원본 ROI에 중심 표시를 띄운다.
        cv2.imshow('line_mask', mask)  # threshold 결과 mask를 띄운다.
        cv2.waitKey(1)  # OpenCV 창을 갱신한다.

        self.cmd_pub.publish(cmd)  # /cmd_vel을 발행한다.

    def destroy_node(self):  # 노드 종료 시 실행된다.
        cv2.destroyAllWindows()  # OpenCV 창을 닫는다.
        super().destroy_node()  # ROS2 노드를 정리한다.


def main(args=None):  # 실행 시작 함수이다.
    rclpy.init(args=args)  # ROS2를 초기화한다.
    node = CameraLineFollow()  # 노드를 생성한다.
    rclpy.spin(node)  # 콜백을 계속 처리한다.
    node.destroy_node()  # 노드를 정리한다.
    rclpy.shutdown()  # ROS2를 종료한다.
```

## 4.3 Camera 예제 3: 화면 중앙 목표물 추종

### 문제의 의의

카메라로 특정 색상 물체를 찾고 화면 중앙에 오도록 조향하면, 사람 추종이나 표식 추종의 기초가 된다. 이 예제는 초록색 목표물을 찾는 간단한 방식으로 작성했다.

```text
→ cv2.imshow()로 인식 결과 확인
```

```python
import numpy as np  # 배열 처리를 위해 사용한다.
import rclpy  # ROS2 Python 라이브러리이다.
from rclpy.node import Node  # 노드 클래스를 가져온다.
from sensor_msgs.msg import CompressedImage  # 압축 이미지 메시지 타입이다.
from geometry_msgs.msg import Twist  # /cmd_vel 메시지 타입이다.

def clamp_steer(value):  # 조향 제한 함수이다.
    return max(-MAX_STEER, min(MAX_STEER, value))  # -0.42 ~ 0.42 범위로 제한한다.


class CameraCenterTargetFollow(Node):  # 색상 목표물 추종 노드이다.
    def __init__(self):  # 노드 생성 시 실행된다.
        super().__init__('camera_center_target_follow')  # 노드 이름을 설정한다.
        self.cmd_pub = self.create_publisher(Twist, '/cmd_vel', 10)  # /cmd_vel Publisher를 만든다.
        self.image_sub = self.create_subscription(CompressedImage, '/camera/color/image_raw/compressed', self.image_callback, 10)  # 압축 이미지 토픽을 구독한다.

    def image_callback(self, msg):  # 이미지가 들어올 때마다 실행된다.
        np_arr = np.frombuffer(msg.data, np.uint8)  # 압축 이미지 데이터를 numpy 배열로 바꾼다.
        frame = cv2.imdecode(np_arr, cv2.IMREAD_COLOR)  # 압축 이미지를 OpenCV 이미지로 복원한다.

        if frame is None:  # 복원 실패 시 처리하지 않는다.
            return  # 콜백을 종료한다.

        height, width, _ = frame.shape  # 이미지 크기를 가져온다.
        hsv = cv2.cvtColor(frame, cv2.COLOR_BGR2HSV)  # 색상 검출을 위해 HSV 색공간으로 바꾼다.

        lower_green = np.array([40, 80, 80])  # 초록색 하한값이다.
        upper_green = np.array([80, 255, 255])  # 초록색 상한값이다.
        mask = cv2.inRange(hsv, lower_green, upper_green)  # 초록색 영역만 흰색으로 만든다.

            cy = int(moments['m01'] / moments['m00'])  # 목표물 y 중심 좌표이다.
            center_x = width // 2  # 화면 x 중심 좌표이다.

            normalized_error = error / center_x  # 오차를 정규화한다.
            steer_cmd = clamp_steer(0.5 * normalized_error)  # 조향 명령을 만들고 제한한다.

            cmd.linear.x = 0.10  # 천천히 전진한다.
            cmd.angular.z = steer_cmd  # 목표물을 중앙에 두기 위한 조향 명령이다.

            cv2.circle(frame, (cx, cy), 10, (0, 0, 255), -1)  # 목표물 중심점을 표시한다.
            cv2.line(frame, (center_x, 0), (center_x, height), (255, 0, 0), 2)  # 화면 중심선을 표시한다.
            self.get_logger().info(f'target cx={cx}, error={error}, steer={steer_cmd:.2f}')  # 상태를 출력한다.
        else:  # 목표물이 없으면 안전하게 정지한다.
            cmd.linear.x = 0.0  # 정지한다.
            cmd.angular.z = clamp_steer(0.0)  # 조향도 0으로 둔다.
            self.get_logger().warn('Target not found. Stop.')  # 경고를 출력한다.

        cv2.imshow('target_view', frame)  # 목표물 중심이 표시된 영상을 보여준다.
        cv2.imshow('target_mask', mask)  # 색상 mask를 보여준다.
        cv2.waitKey(1)  # OpenCV 창을 갱신한다.

        self.cmd_pub.publish(cmd)  # /cmd_vel을 발행한다.

    def destroy_node(self):  # 노드 종료 시 실행된다.
        cv2.destroyAllWindows()  # OpenCV 창을 닫는다.
        super().destroy_node()  # ROS2 노드를 정리한다.
```

## 5.2 Odom 예제 2: `/odom` 기준 1m 전진 후 정지

### 문제의 의의

센서 기반 제어에서는 단순히 시간을 기준으로 움직이는 것보다, 실제 이동량을 기준으로 제어하는 것이 더 안정적이다. 이 예제는 `/odom`의 위치값을 이용해 시작 위치에서 1m 이동하면 정지한다.

### 코드 흐름

```text
/odom 수신
→ 첫 위치를 시작 위치로 저장
→ 현재 위치와 시작 위치의 거리 계산
→ 1m 미만이면 전진
→ 1m 이상이면 정지
```

파일명: `sensor_cmd_examples/odom_move_1m_stop.py`

```python
import math  # 거리 계산에 사용한다.
import rclpy  # ROS2 Python 라이브러리이다.
from rclpy.node import Node  # 노드 클래스를 가져온다.
from nav_msgs.msg import Odometry  # /odom 메시지 타입이다.
from geometry_msgs.msg import Twist  # /cmd_vel 메시지 타입이다.

MAX_STEER = 0.42  # LIMO Ackermann angular.z 제한값이다.


def clamp_steer(value):  # 조향 제한 함수이다.
    return max(-MAX_STEER, min(MAX_STEER, value))  # -0.42 ~ 0.42 범위로 제한한다.


class OdomMove1mStop(Node):  # 1m 전진 후 정지하는 노드이다.
    def __init__(self):  # 노드 생성 시 실행된다.
        super().__init__('odom_move_1m_stop')  # 노드 이름을 설정한다.
        self.start_x = None  # 시작 x 위치를 저장한다.
        self.start_y = None  # 시작 y 위치를 저장한다.
        self.finished = False  # 이동 완료 여부를 저장한다.
        self.cmd_pub = self.create_publisher(Twist, '/cmd_vel', 10)  # /cmd_vel Publisher를 만든다.
        self.odom_sub = self.create_subscription(Odometry, '/odom', self.odom_callback, 10)  # /odom을 구독한다.

    def odom_callback(self, msg):  # Odom 메시지가 들어올 때마다 실행된다.
        x = msg.pose.pose.position.x  # 현재 x 위치를 읽는다.
        y = msg.pose.pose.position.y  # 현재 y 위치를 읽는다.

        if self.start_x is None:  # 시작 위치가 아직 없다면 초기화한다.
            self.start_x = x  # 현재 x를 시작 x로 저장한다.
            self.start_y = y  # 현재 y를 시작 y로 저장한다.

        dx = x - self.start_x  # 시작 위치에서 x 방향 이동량을 계산한다.
        dy = y - self.start_y  # 시작 위치에서 y 방향 이동량을 계산한다.
        distance = math.sqrt(dx * dx + dy * dy)  # 시작 위치로부터 이동 거리를 계산한다.

        cmd = Twist()  # 속도 명령 메시지를 만든다.

        if distance >= 1.0:  # 이동 거리가 1m 이상이면 목표에 도달한 것이다.
            cmd.linear.x = 0.0  # 정지한다.
            cmd.angular.z = clamp_steer(0.0)  # 조향도 0으로 둔다.
            self.finished = True  # 완료 상태로 저장한다.
            self.get_logger().info(f'Arrived. distance={distance:.2f}m')  # 도착 상태를 출력한다.
        elif not self.finished:  # 아직 1m에 도달하지 않았다면 계속 전진한다.
            cmd.linear.x = 0.12  # 천천히 전진한다.
            cmd.angular.z = clamp_steer(0.0)  # 직진한다.
            self.get_logger().info(f'Moving. distance={distance:.2f}m')  # 이동 상태를 출력한다.

        self.cmd_pub.publish(cmd)  # /cmd_vel을 발행한다.


def main(args=None):  # 실행 시작 함수이다.
    rclpy.init(args=args)  # ROS2를 초기화한다.
    node = OdomMove1mStop()  # 노드를 생성한다.
    rclpy.spin(node)  # 콜백을 계속 처리한다.
    node.destroy_node()  # 노드를 정리한다.
    rclpy.shutdown()  # ROS2를 종료한다.
```

## 8. 수업 진행 추천 순서

1. `ros2 topic list`로 `/scan`, `/imu`, `/camera/color/image_raw/compressed`, `/odom`, `/cmd_vel`이 있는지 확인한다.
2. `ros2 topic echo /scan --once`로 LiDAR 메시지 구조를 본다.
3. `ros2 topic echo /imu --once`로 IMU quaternion 구조를 본다.
4. `ros2 topic echo /odom --once`로 위치와 속도 구조를 본다.
5. LiDAR 전방 정지 예제로 센서 기반 정지를 실습한다.
6. `enumerate()` 각도 계산 예제로 LiDAR index와 실제 각도의 관계를 이해한다.
7. IMU yaw 유지 예제로 quaternion → yaw → 조향 보정 흐름을 이해한다.
8. 카메라 예제에서 `cv2.imshow()`로 전처리 결과를 직접 확인한다.
9. 마지막으로 `/odom` 기반 1m 이동 후 정지를 실습한다.

## 9. 핵심 정리

```text
LiDAR는 거리 기반 안전 판단에 좋다.
IMU는 방향과 기울기 판단에 좋다.
Camera는 라인, 색상, 목표물 인식에 좋다.
Odom은 실제 이동량과 현재 속도 확인에 좋다.
/cmd_vel은 센서 판단 결과를 로봇 행동으로 바꾸는 제어 토픽이다.
```

LIMO Ackermann 모드에서는 다음 원칙을 지킨다.

```text
제자리 회전 금지
전진하면서 조향
linear.x 사용
angular.z 사용
angular.z는 반드시 -0.42 ~ 0.42로 제한
```
