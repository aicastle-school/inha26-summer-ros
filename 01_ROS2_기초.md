## 3. ROS2 워크스페이스 및 패키지 관리

### 3.1 워크스페이스란?

ROS2 워크스페이스는 여러 ROS2 패키지를 모아서 개발하고 빌드하는 작업 공간이다.

일반적으로 하나의 워크스페이스 안에는 다음과 같은 폴더가 있다.

```text
physicar_ws/
  ├── src/          # 소스 코드 디렉토리
  ├── build/        # 빌드 중간 결과물
  ├── install/      # 실행 가능한 설치 결과물
  └── log/          # 빌드 로그
```

## 5.2 미니멀 퍼블리셔

미니멀 퍼블리셔는 ROS2 토픽으로 데이터를 계속 보내는 가장 기본적인 예제이다.
이 예제에서는 `std_msgs/msg/String` 타입 메시지를 만들고, `topic`이라는 이름의 토픽으로 0.5초마다 문자열을 발행한다.

```text
노드 실행 → 퍼블리셔 생성 → 타이머 생성 → 주기적으로 메시지 발행
```

### 5.2.1 파일 생성

```bash
cd ~/physicar_ws/src/my_first_package/my_first_package
nano minimal_publisher.py
```

### 5.2.2 코드

```python
import rclpy
from rclpy.node import Node
from std_msgs.msg import String

class MinimalPublisher(Node):
    def __init__(self):
        super().__init__('minimal_publisher')
        self.publisher_ = self.create_publisher(String, 'topic', 10)
        timer_period = 0.5  # seconds
        self.timer = self.create_timer(timer_period, self.timer_callback)
        self.i = 0

    def timer_callback(self):
        msg = String()
        msg.data = 'Hello World: %d' % self.i
        self.publisher_.publish(msg)
        self.get_logger().info('Publishing: "%s"' % msg.data)
        self.i += 1

def main(args=None):
    rclpy.init(args=args)
    minimal_publisher = MinimalPublisher()
    rclpy.spin(minimal_publisher)
    minimal_publisher.destroy_node()
    rclpy.shutdown()

if __name__ == '__main__':
    main()
```

### 5.2.3 코드 설명

이 예제는 `topic`이라는 토픽으로 문자열 메시지를 반복해서 발행하는 퍼블리셔 노드이다.

| 코드 | 의미 |
|---|---|
| `create_publisher(String, 'topic', 10)` | `String` 타입 메시지를 `topic` 토픽으로 발행하는 퍼블리셔 생성 |
| `create_timer(0.5, self.timer_callback)` | 0.5초마다 `timer_callback()` 함수 실행 |
| `msg = String()` | 발행할 메시지 객체 생성 |
| `msg.data = ...` | 문자열 데이터 입력 |
| `self.publisher_.publish(msg)` | 토픽으로 메시지 발행 |

즉, 이 노드는 0.5초마다 `Hello World: 0`, `Hello World: 1` 같은 문자열을 계속 발행한다.

## 5.3 미니멀 서브스크라이버

미니멀 서브스크라이버는 ROS2 토픽으로 들어오는 데이터를 받아서 처리하는 가장 기본적인 예제이다.
퍼블리셔가 `topic` 토픽에 문자열을 발행하면, 서브스크라이버는 같은 `topic`을 구독하고 메시지가 들어올 때마다 콜백 함수를 실행한다.

```text
노드 실행 → 서브스크라이버 생성 → 토픽 대기 → 메시지 수신 시 콜백 실행
```

### 5.3.1 파일 생성

```bash
cd ~/physicar_ws/src/my_first_package/my_first_package
nano minimal_subscriber.py
```

### 5.3.2 코드

```python
import rclpy
from rclpy.node import Node
from std_msgs.msg import String

class MinimalSubscriber(Node):
    def __init__(self):
        super().__init__('minimal_subscriber')
        self.subscription = self.create_subscription(
            String,
            'topic',
            self.listener_callback,
            10)
        self.subscription

    def listener_callback(self, msg):
        self.get_logger().info('I heard: "%s"' % msg.data)

def main(args=None):
    rclpy.init(args=args)
    minimal_subscriber = MinimalSubscriber()
    rclpy.spin(minimal_subscriber)
    minimal_subscriber.destroy_node()
    rclpy.shutdown()

if __name__ == '__main__':
    main()
```

### 5.3.3 코드 설명

이 예제는 `topic` 토픽에서 문자열 메시지를 받아서 터미널에 출력하는 서브스크라이버 노드이다.

| 코드 | 의미 |
|---|---|
| `create_subscription(String, 'topic', self.listener_callback, 10)` | `topic` 토픽을 구독하고 메시지가 오면 `listener_callback()` 실행 |
| `listener_callback(self, msg)` | 메시지를 받았을 때 자동으로 호출되는 함수 |
| `msg.data` | 수신한 문자열 데이터 |
| `get_logger().info(...)` | 터미널에 수신 결과 출력 |

즉, 퍼블리셔가 `topic`으로 메시지를 보내면 이 노드가 그 메시지를 받아서 `I heard: ...` 형태로 출력한다.

## 5.5 PhysiCar 간단한 주행 코드

### 5.5.1 파일 생성

### 5.5.2 코드

```python
import rclpy
from rclpy.node import Node
from geometry_msgs.msg import Twist
import time

class PhysiCarDriveController(Node):
    def __init__(self):
        super().__init__('drive_controller')
        self.cmd_vel_pub = self.create_publisher(Twist, '/cmd_vel', 10)
        self.get_logger().info('PhysiCar Drive Controller initialized')

    def move_forward(self, speed=0.5, duration=3.0):
        msg = Twist()
        msg.linear.x = float(speed)

        start_time = time.time()
        self.get_logger().info(f'Moving forward: speed={speed}')
        while time.time() - start_time < duration:
            self.cmd_vel_pub.publish(msg)
            time.sleep(0.1)

    def move_backward(self, speed=0.3, duration=2.0):
        msg = Twist()
        msg.linear.x = -float(speed)

        start_time = time.time()
        self.get_logger().info(f'Moving backward: speed={speed}')
        while time.time() - start_time < duration:
            self.cmd_vel_pub.publish(msg)
            time.sleep(0.1)

    def stop(self):
        msg = Twist()
        msg.linear.x = 0.0
        msg.angular.z = 0.0
        self.cmd_vel_pub.publish(msg)
        self.get_logger().info('Stopped')
        time.sleep(0.2)

def main(args=None):
    rclpy.init(args=args)
    controller = PhysiCarDriveController()

    try:
        controller.move_forward(speed=0.5, duration=3.0)
        time.sleep(1.0)

        controller.move_backward(speed=0.3, duration=2.0)
        time.sleep(1.0)

        controller.stop()

        controller.get_logger().info('Drive sequence completed')
    except KeyboardInterrupt:
        controller.get_logger().info('Interrupted by user')
    finally:
        controller.destroy_node()
        rclpy.shutdown()

if __name__ == '__main__':
    main()
```

### 5.6.2 코드

```python
import rclpy
from rclpy.node import Node
from sensor_msgs.msg import CompressedImage
import cv2
import numpy as np

class CameraViewer(Node):
    def __init__(self):
        super().__init__('camera_viewer')
        self.subscription = self.create_subscription(
            CompressedImage,
            '/camera/image_raw/compressed',
            self.image_callback,
            10)
        self.frame_count = 0
        self.get_logger().info('Camera Viewer Node started. Window will appear shortly.')

    def image_callback(self, msg):
        self.frame_count += 1

        try:
            np_arr = np.frombuffer(msg.data, np.uint8)
            cv_image = cv2.imdecode(np_arr, cv2.IMREAD_COLOR)

            if self.frame_count % 100 == 0:
                height, width = cv_image.shape[:2]
                self.get_logger().info(f'Frame {self.frame_count}: {width}x{height}')

            cv2.imshow("PhysiCar Camera View", cv_image)
            cv2.waitKey(1)

        except Exception as e:
            self.get_logger().error(f'Error processing image: {e}')

    def destroy_node(self):
        cv2.destroyAllWindows()
        super().destroy_node()

def main(args=None):
    rclpy.init(args=args)
    camera_viewer = CameraViewer()

    try:
        rclpy.spin(camera_viewer)
    except KeyboardInterrupt:
        camera_viewer.get_logger().info('Node stopped by user')
    finally:
        camera_viewer.destroy_node()
        rclpy.shutdown()

if __name__ == '__main__':
    main()
```

## 5.7 launch 파일 만들기

### 5.7.1 launch 파일이란?

launch 파일은 여러 ROS2 노드를 한 번에 실행하거나, 노드 실행 옵션을 정리해서 관리할 때 사용하는 실행 설정 파일이다.

예를 들어 매번 다음처럼 여러 명령어를 직접 실행하는 대신,

```bash
ros2 run my_first_package basic_node
ros2 run my_first_package talker
ros2 run my_first_package listener
```

launch 파일 하나로 여러 노드를 한 번에 실행할 수 있다.

### 5.7.3 기본 launch 파일 작성

`basic_launch.py` 파일에 다음 내용을 작성한다.

```python
from launch import LaunchDescription
from launch_ros.actions import Node


def generate_launch_description():
    return LaunchDescription([
        Node(
            package='my_first_package',
            executable='basic_node',
            name='my_basic_node'
        )
    ])
```

각 항목의 의미는 다음과 같다.

| 항목 | 의미 |
|---|---|
| `LaunchDescription` | 실행할 노드 목록을 담는 launch 설정 객체 |
| `Node` | launch 파일에서 실행할 ROS2 노드 |
| `package` | 실행할 노드가 들어있는 패키지 이름 |
| `executable` | `setup.py`의 `console_scripts`에 등록한 실행 이름 |
| `name` | 실행될 노드의 이름 |

---

### 5.7.4 setup.py에 launch 파일 설치 설정 추가

Python 기반 ROS2 패키지에서는 launch 파일을 만들기만 하면 안 된다.

`ros2 launch` 명령어가 launch 파일을 찾을 수 있도록 `setup.py`의 `data_files`에 launch 폴더 설치 설정을 추가해야 한다.

먼저 `setup.py`를 연다.

```bash
cd ~/physicar_ws/src/my_first_package
nano setup.py
```

파일 상단에 다음 import를 추가한다.

```python
import os
from glob import glob
```

그리고 `data_files` 부분에 다음 줄을 추가한다.

```python
(os.path.join('share', package_name, 'launch'), glob('launch/*.py')),
```

예시는 다음과 같다.

```python
import os
from glob import glob
from setuptools import find_packages, setup

package_name = 'my_first_package'

setup(
    name=package_name,
    version='0.0.0',
    packages=find_packages(exclude=['test']),
    data_files=[
        ('share/ament_index/resource_index/packages',
            ['resource/' + package_name]),
        ('share/' + package_name, ['package.xml']),
        (os.path.join('share', package_name, 'launch'), glob('launch/*.py')),
    ],
    install_requires=['setuptools'],
    zip_safe=True,
    maintainer='user',
    maintainer_email='user@example.com',
    description='ROS2 basic practice package',
    license='Apache-2.0',
    entry_points={
        'console_scripts': [
            'basic_node = my_first_package.basic_node:main',
        ],
    },
)
```

### 5.7.6 여러 노드를 한 번에 실행하는 launch 파일

from launch_ros.actions import Node


def generate_launch_description():
    return LaunchDescription([

            executable='talker',
            name='minimal_publisher'

실행하면 `talker`와 `listener` 노드가 동시에 실행된다.

# 9. 주요 학습 포인트

## 9.1 ROS2의 기본 구성요소

### 9.1.1 노드

노드는 ROS2의 기본 실행 단위이다.

- 독립적으로 실행되는 프로세스
- 다른 노드와 통신 가능
- 센서 처리, 제어, 인식, 판단 등을 각각의 노드로 나눌 수 있음

---

### 9.1.2 토픽

토픽은 노드 간 비동기 통신 채널이다.

- 퍼블리셔: 데이터 발행
- 서브스크라이버: 데이터 수신
- 1:N, N:1, N:N 통신 가능

비유하면 토픽은 여러 사람이 볼 수 있는 설문지 또는 게시판과 비슷하다.

---

### 9.1.3 서비스

서비스는 노드 간 동기 요청-응답 통신 방식이다.

- 클라이언트: 요청
- 서버: 응답
- 결과를 기다려야 하는 명령에 적합

---

### 9.1.4 파라미터

파라미터는 노드의 런타임 설정값이다.

- 속도 제한
- 센서 옵션
- 알고리즘 설정값
- 노드 실행 중 일부 값 변경 가능

---
